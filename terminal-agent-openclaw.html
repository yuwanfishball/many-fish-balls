🤖 2026智源大会 · 终端智能体与OpenClaw

📖 论坛简介

本论坛围绕 「大模型智能体如何获得真实执行能力」 展开，核心关键词是 Harness（连接/驾驭）体系——通过 Harness 框架将模型、工具、环境、记忆、符号结构、安全沙箱、端云系统有机整合，使 Agent 从单纯的"会聊天的语言模型"进化为能在终端设备和组织场景中持续完成任务的执行系统。

刘知远教授指出：OpenClaw 的历史意义不在于它是否长存，而在于它标志着大模型智能体从云端聊天进入真实工作场域——正如浏览器没有发明互联网，却定义了普通人进入互联网的方式。论坛集结产学研领域专家学者，从系统架构、移动端集成、神经符号融合、环境构建、产品落地等多维度展开深入研讨。

▶ 5:35 – 40:35 开场致辞 & 主题报告：从 OpenClaw 看智能体技术发展趋势 刘知远 — 清华大学计算机系教授，面壁智能创始人

刘知远（清华大学计算机系长聘教授、面壁智能创始人、论坛主席）致开场辞并做主题报告：

  • OpenClaw 标志着大模型智能体从「云端聊天助手」进入真实工作场域
  • 核心命题：谁能定义 Agent 时代的「操作层」？
  • Agent 正从「工具使用者」走向「环境参与者」，从「任务执行器」走向「持续学习体」
  • 端侧智能的三大命题：端侧模型最重要的使命？落地形态？谁能留在牌桌上？

核心观点

OpenClaw 被比作「Agent 时代的浏览器」——并未发明底层技术，但第一次把分散能力组织成可被大众感知的形态。正如浏览器定义了普通人进入互联网的方式，OpenClaw 正在定义 Agent 进入工作场景的方式。
▶ 40:35 – 1:05:35 从 OpenClaw 看智能体技术发展趋势 林衍凯 — 中国人民大学高瓴人工智能学院准聘副教授

林衍凯（中国人民大学高瓴人工智能学院准聘副教授）系统分析 OpenClaw 背后的技术架构与趋势：

  • OpenClaw 的本质是系统集成——把模型、Skill、记忆、接口和用户入口组织到一起
  • 智能体技术栈的分层：模型层 → 能力层 → 交互层 → 应用层
  • 当前瓶颈不在模型能力，而在系统架构的整合效率
  • 未来趋势：从单 Agent 到多 Agent 协作，从固定流程到自适应编排

核心观点

Agent 框架的竞争本质是「系统集成效率」的竞争——谁能用更少的工程成本，把更强的模型能力转化为更可靠的任务执行，谁就掌握了 Agent 时代的入口。
▶ 1:05:35 – 1:30:35 Organization Model：让组织变得可生成 钱忱 — 上海交通大学人工智能学院副教授

钱忱（上海交通大学人工智能学院副教授）提出「可生成组织」概念：

  • 提出「可生成组织」（Generative Organization）概念——多个 Agent 如何自动生成组织结构
  • 核心机制：角色分配、任务分解、协作流程的自动化生成
  • 扩展到教育场景：TeachMaster 教学智能体、Agentic University 智能大学构想
  • 组织模型的三个层次：个体 Agent → 团队协作 → 组织级智能

核心观点

当 Agent 不仅能执行单个任务，还能自动组建「虚拟团队」完成复杂项目时，教育、科研、企业管理的组织形态将被根本性重塑。
▶ 1:30:35 – 1:55:35 Mobile Agent Harness（移动智能体集成框架） 李元春 — 清华大学智能产业研究院助理教授

李元春（清华大学智能产业研究院助理教授）提出核心框架 Agent = Model + Harness：

  • 端侧 Agent 的四大基础设施：
    • APP 知识库——让 Agent 理解每个应用的能力边界
    • 自动探索机制——Agent 自主发现和适配新环境
    • 状态转换图——对复杂任务流程的结构化建模
    • 合成环境——弥补真实环境数据的不足
  • 移动端 Agent 的独特挑战：屏幕理解、触控操作、跨应用协同

核心观点

没有好的 Harness 体系，再强的模型也无法在终端设备上可靠执行任务。Harness 是连接模型能力与真实世界的「最后一公里」。
▶ 1:55:35 – 2:20:35 Neuro-Symbolic Agent（神经符号智能体） 郭兰哲 — 南京大学智能科学与技术学院准聘助理教授

郭兰哲（南京大学智能科学与技术学院准聘助理教授）从神经符号角度重新审视 Agent 架构：

  • Agent 需要的五种关键能力：
    • 状态判断——准确感知当前执行进度
    • 变量绑定——将抽象指令映射到具体操作
    • 控制流——条件分支、循环、异常处理
    • 失败修复——出错后自主诊断和恢复
    • 可执行图结构——将经验压缩为可操作、可泛化的符号结构
  • 从反应式决策到可验证决策：神经网络的灵活性 + 符号系统的可靠性

核心观点

纯神经网络的黑盒决策在关键任务中不可靠。神经符号方法将 Agent 的「直觉」与「逻辑」结合，是实现可信自主智能体的关键路径。
▶ 2:20:35 – 2:40:35 Environment is the Missing "Data" for Agents 范文栋 — CAMEL 技术负责人

范文栋（CAMEL 技术负责人）提出核心论点：环境是 Agent 缺失的关键「数据」：

  • 没有可规模化、可验证的环境，Agent 无法形成真正的长程能力
  • CAMEL 在环境构建方面的实践：
    • 合成任务环境——大规模生成训练场景
    • 对抗性环境——测试 Agent 的鲁棒性边界
    • 多 Agent 社会环境——模拟真实协作与竞争
  • 环境即数据：环境的质量决定了 Agent 能力的上限

核心观点

我们花了太多精力优化模型，却忽略了 Agent 赖以成长的「土壤」——环境。没有丰富、可扩展的环境基础设施，Agent 的进化将撞上数据天花板。
▶ 2:40:35 – 3:00:35 AI 原生战略下的 Agent 产品思考与实践 段亦涛 — 网易有道首席科学家

段亦涛（网易有道首席科学家）分享教育场景下 Agent 产品的落地经验：

  • 核心原则：围绕业务目标优化，而非论文 Benchmark
  • 教育 Agent 的独特需求：
    • 可解释性——学生和家长需要理解 AI 的决策
    • 个性化——适应不同学习风格和进度
    • 安全性——教育场景对内容安全要求极高
  • AI 原生战略：不是给旧产品加 AI 功能，而是从 AI 能力出发重新设计产品

核心观点

「先找到用户愿意付费的真实需求，再用 Agent 能力去满足它」——产业落地的关键是业务价值驱动，而非技术炫技。
▶ 3:00:35 – 3:20:35 从"能聊"到"能力"：让智能体拥有超级执行力 李景秋 — 百度搭子 DuMate 产品总架构师

李景秋（百度搭子 DuMate 产品总架构师）阐述从「能聊」到「能力」的关键跃迁：

  • 超级执行力的三大支柱：
    • 意图理解——从模糊需求到精确任务描述
    • 工具编排——自动选择和组合合适的工具链
    • 执行闭环——任务 → 反馈 → 调整 → 完成
  • 百度 DuMate 的实践：如何让 Agent 真正「干活」而不是「聊天」
  • 端侧 Agent 的产品化挑战：延迟、功耗、隐私

核心观点

用户不会为「会聊天的 AI」付费，但会为「能帮我把事办成的 AI」付费。执行力是 Agent 产品从玩具走向工具的必经之路。
▶ 3:20:35 – 3:40:35 Work Buddy 思考与实践 汪晟杰 — 腾讯云资深产品专家

汪晟杰（腾讯云资深产品专家、Work Buddy 产品负责人、本场主持人）提出 AgentOS 概念：

  • AgentOS 概念——面向 Agent 的新型操作系统范式
  • Work Buddy 的核心架构：
    • 端云协同——端侧低延迟 + 云端强算力
    • 安全沙箱——Agent 执行环境隔离，防止越权操作
    • 审计追踪——每一步操作可记录、可回溯
    • 记忆分层——短期工作记忆 + 长期知识积累
    • 多设备调度——跨手机、PC、平板的统一 Agent 体验
  • 腾讯在办公场景下的 Agent 落地实践

核心观点

Agent 的操作系统不是 Windows/macOS 的简单延伸——它需要重新定义进程（Agent 生命周期）、文件系统（记忆管理）、权限模型（能力边界）和安全模型（行为审计）。
